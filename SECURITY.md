# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| `main` | Yes |
| `2.x` | Yes |
| `< 2.0` | No |

## Reporting a Vulnerability

Please do **not** open public GitHub issues for security vulnerabilities.

1. Use GitHub's private vulnerability reporting for this repository if it is enabled.
2. If private reporting is not available, contact the maintainer,
   [@armaghan-work](https://github.com/armaghan-work), through the contact options
   on the GitHub profile.
3. Include the affected version or branch, reproduction steps, impact, and any
   suggested mitigation if you have one.

An acknowledgement should be provided within 5 business days. Confirmed issues
will be fixed and disclosed responsibly.
